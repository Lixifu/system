## 问题分析
组织方仪表盘显示已报名志愿者数量为0，而实际有志愿者报名。通过测试发现，问题出在`backend/routes/user.js`文件中的唯一志愿者数量计算逻辑。

## 解决方案
修改`backend/routes/user.js`文件中的`/dashboard/organizer`端点，将复杂的COUNT(DISTINCT)查询替换为更可靠的GROUP BY查询：

1. **原代码**：使用`sequelize.fn('COUNT', sequelize.fn('DISTINCT', sequelize.col('user_id')))`的复杂查询，返回结果为0
2. **修复后代码**：使用`ActivityParticipant.findAll()`配合`group: ['user_id']`，然后获取结果数组长度，正确返回唯一志愿者数量

## 修复代码
```javascript
// 然后统计这些活动中不重复的唯一用户数量
let totalRegisteredVolunteers = 0;
if (activityIds.length > 0) {
    // 使用GROUP BY和COUNT来统计唯一用户
    const uniqueVolunteers = await ActivityParticipant.findAll({
        attributes: ['user_id'],
        where: {
            activityId: activityIds
        },
        group: ['user_id']
    });
    totalRegisteredVolunteers = uniqueVolunteers.length;
}
```

## 验证结果
- 修复后API返回正确的已报名志愿者数量2
- 活动列表中每个活动的已报名人数显示正确
- 组织方仪表盘数据与实际情况一致
